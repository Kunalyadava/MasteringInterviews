OOP in Angular & Inheritance

What is Object-Oriented Programming (OOP) in Angular?
Angular is built on TypeScript, which supports Object-Oriented Programming (OOP) principles. OOP in Angular is about organizing code using classes, objects, inheritance, encapsulation, polymorphism, and abstraction. Angular components, services, and directives are all based on classes and methods, which makes OOP a core concept in building Angular applications.
Key OOP Concepts in Angular:
  • Classes: Used to define Angular components, services, and directives.
  • Encapsulation: Data and behavior are encapsulated in Angular classes (components, services).
  • Inheritance: Allows one class to inherit from another, enabling code reuse and specialization.
  • Polymorphism: Enables objects of different types to be treated as instances of the same class, enhancing flexibility.
  • Abstraction: Hides complex implementation details and exposes only necessary functionality.
What is Inheritance in Angular?
Inheritance in Angular allows a class (child class) to inherit properties and methods from another class (parent class). This is useful for code reuse and extending functionality without duplicating code. In Angular, inheritance is commonly used with components, services, and directives to build specialized versions of a base class.
Example of Inheritance in Angular (Component Inheritance):
Here is an example of a parent class and a child class in Angular, demonstrating inheritance. 

// Parent class Component
import { Component } from '@angular/core';

@Component({
  selector: 'app-base',
  template: `
Base Component
`
})
export class BaseComponent {
  message = "Hello from the base component!";

  showMessage() {
    console.log(this.message);
  }
}

// Child class Component extending the parent
@Component({
  selector: 'app-child',
  template: `
Child Component
`
})
export class ChildComponent extends BaseComponent {
  constructor() {
    super();
    this.message = "Hello from the child component!";
  }

  // Overriding the method from BaseComponent
  showMessage() {
    console.log(`Child's Message: ${this.message}`);
  }
}
In this example:
  • BaseComponent: This is the parent class with a message and method to show it.
  • ChildComponent: This class extends the `BaseComponent`, overriding the `showMessage()` method to customize its behavior.
The child class has access to the parent class's properties and methods, and can override them as needed.
Benefits of Inheritance in Angular:
  • Code Reusability: Child classes can reuse code from parent classes without duplication.
  • Maintainability: Inheritance allows you to extend and customize behavior without modifying existing code.
  • Modularity: Helps in creating a modular code structure where components and services can be extended as needed.
  • Clean Structure: Code becomes more organized, making it easier to manage large applications.
How is Inheritance Used in Angular Services?
Angular services can also benefit from inheritance. You can create a base service with shared functionality (e.g., logging, data handling), and then extend this service in other services that need more specific behavior. 

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class BaseService {
  log(message: string) {
    console.log('Base Log: ' + message);
  }
}

@Injectable({
  providedIn: 'root',
})
export class ChildService extends BaseService {
  log(message: string) {
    console.log('Child Log: ' + message);  // Overridden method
  }

  fetchData() {
    this.log('Fetching data...');
  }
}
In this example:
  • BaseService: A base service providing a simple log method.
  • ChildService: A service that extends `BaseService`, overrides the log method, and adds new functionality like `fetchData`.
This is useful for creating specialized services while still leveraging the common behavior in the base service.
