RxJS Questions and Answers

Question 1: What is the full form of RxJS?
RxJS stands for Reactive Extensions for JavaScript. It's a library for composing asynchronous and event-based programs by using observable sequences.
Question 2: What is the purpose of RxJS?
RxJS allows you to handle asynchronous data streams and events. It provides a set of operators and observables to make it easy to work with complex data flows, particularly useful in front-end frameworks like Angular where reactive programming patterns are beneficial for handling UI events, HTTP requests, and more.
Question 3: What are observables and observers?
  • Observables: These are data producers that can emit values over time (similar to streams). Observables are lazy, meaning they do not emit any values until they are subscribed to.
  • Observers: These are data consumers, essentially the objects or functions that listen to values emitted by the Observable. Observers can handle emitted values, errors, and completion signals from Observables.
Question 4: Explain the use of subscribe with sample code
The subscribe method allows you to execute logic each time an Observable emits a value.
import { Observable } from 'rxjs'; const observable = new Observable(subscriber => { subscriber.next('Hello'); subscriber.next('World'); subscriber.complete(); }); observable.subscribe({ next(value) { console.log(value); }, error(err) { console.error('Error:', err); }, complete() { console.log('Completed'); } }); // Output: // Hello // World // Completed
Question 5: How to unsubscribe in RxJS?
To stop receiving values from an Observable, you need to unsubscribe, which is particularly useful to prevent memory leaks.
const subscription = observable.subscribe(value => console.log(value)); // Unsubscribe when you no longer need the subscription subscription.unsubscribe();
Question 6: Explain the concept of operators with sample code
Operators in RxJS are functions that manipulate data streams. Common operators include map, filter, and mergeMap.
import { of } from 'rxjs'; import { map, filter } from 'rxjs/operators'; const observable = of(1, 2, 3, 4, 5).pipe( filter(value => value % 2 === 0), map(value => value * 10) ); observable.subscribe(value => console.log(value)); // Output: // 20 // 40
Question 7: How to install RxJS?
To install RxJS, use npm:
npm install rxjs
Question 8: Differentiate between Promise and RxJS
  • Promises: A Promise handles a single asynchronous event (resolved once). Once created, it cannot be canceled.
  • RxJS (Observables): Observables handle a stream of events over time, can emit multiple values, and support operators to transform data streams. Observables are cancellable (using unsubscribe).
Question 9: In Angular, where have you used RxJS?
RxJS is commonly used in Angular for:
  • HTTP requests: Observables handle responses from the HttpClient.
  • Event handling: Handling UI events reactively.
  • Form handling: Reactive forms use Observables to manage form controls.
  • State management: Handling state updates and communication between components.
Question 10: Which operators have you used from RxJS?
Commonly used operators include map, filter, switchMap, mergeMap, concatMap, take, catchError, and debounceTime.
